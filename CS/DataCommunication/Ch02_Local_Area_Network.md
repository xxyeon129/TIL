# Ch02: Local Area Network

# 01 LAN의 개요

## ✅ LAN의 정의

**제한된 거리에 있는 / 다수의 (독립된) 컴퓨터 기기들이 / 상호 간에 통신이 가능하도록 하는 / 데이터 통신 시스템**

**소규모 지역에서 장치들을 연결하는 네트워크 방식**. 주로 학교, 회사, 집처럼 한정된 공간 내에서 여러 컴퓨터나 장치들이 통신할 수 있도록 구성됨.

Kenneth와 Freeman의 정의

- LAN은 **단일 기관의 소유**일 것
- 수 마일 범위 이내에 **지역적으로 한정**되어 있을 것
- 어떤 종류의 **스위칭 기술**을 갖고 있을 것
- 원거리 네트워크(광역 네트워크)의 경우보다 **높은 통신 속도**를 가질 것(근거리이기 때문에)

> [!NOTE]
>
> - 스위칭: 회선교환방식, 패킷교환방식 두 가지 방식
> - LAN 표준 속도: 10Mbps

## ✅ LAN의 특징

### ✨ LAN의 이점

- 상대적으로 **낮은 비용**이 드는 매체로 **높은 대역폭 제공**이 가능하다. → **빠른 데이터 전송** 가능
- Plug-In 연결만으로 **네트워크 확장, 단말 장치의 이동/변경이 가능**하다.
    - 사무실에 새 컴퓨터를 추가할 때 간단히 네트워크 케이블을 연결하거나, 무선 네트워크로 접속할 수 있음
- 다양한 응용을 수용할 수 있으며, **많은 수의 장치를 연결**할 수 있다.
    - 프린터, 파일 공유, 인터넷 연결 등 다양한 서비스 제공
    - 한 네트워크에 여러 개의 장치(프린터, 스마트폰 등) 쉽게 연결 가능
- 게이트웨이, 브리지, *라우터 등의 네트워킹 장비들을 이용하여 **다른 네트워크와 연동**이 가능하다.
- **중앙**에 집중되어 있는 컴퓨팅 **자원**을 가장 편리한 장소에 **분배**하여 위치시킬 수 있다.
- 하나의 **중앙** 지점에서 **네트워크에 대한 모니터링**이 가능하기 때문에 / 네트워크의 상태나 자원 사용률 등을 보장하기 위한 **네트워크 관리가 용이**한 편이다.
- **오랜 기간 사용으로 LAN 기술 자체가 검증되었으며**, 숙련된 LAN 기술자가 많기 때문에 **구축 및 운영, 유지가 편리**하다.

> [!NOTE]
>
> \*라우터: 경로 설정하는 데 쓰임(패킷 전송)
>
> \*\*NMS: Network Monitoring System

</aside>

### 💩 LAN의 단점

- LAN은 **짧은 거리에서의 통신만**을 지원하므로, **거리를 확장**하기 위해서는 리피터, 허브 혹은 브리지와 같은 **네트워킹 장비를 사용**해야 한다.
- 매체에 대한 접근 제어 방식으로 **CSMA/CD**를 사용할 경우에는 한 매체에 연결되는 스테이션의 수가 한정된다. (**한 네트워크에 연결할 수 있는 장치 수 제한**)

> [!NOTE]
> 거리 확장 → 길이 길어짐 → 저항값 올라감 → 리피터(신호재생기; 신호 감쇄된 것을 다시 재생해줌) 장비 필요

## ✅ IEEE 802 표준: Data Link 계층

많은 네트워크 장비 공급 업체들로부터 출시되는 관련 **장비들 간의 호환성 및 장비 간 연동**을 위해 IEEE(미국 전기전자 학회)는 802 위원회를 조직해 LAN에 관한 표준화 추진

> [!NOTE]
>
> IEEE 802 참조모델과 OSI 참조 모델을 같이 사용함
>
> IEEE 802 참조 모델과 OSI 참조 모델의 차이점: Data Link Layer가 IEEE 802의 경우 두 개(Logical Link Control, Medium Access Control)로 나뉘어져 있음 이걸 제외하면 동일

### ✔️ LLC (Logical Link Control; 논리연결제어) 부계층

- 데이터의 **흐름 제어**와 **에러 검출**을 담당
    - 흐름 제어: 데이터가 너무 빠르게 전송되지 않도록 제어하여 네트워크 혼잡 방지
- Data Link 계층의 상위 계층인 **`네트워크 계층`과 `LAN의 MAC 계층`을 연결해주는 인터페이스**
    - 네트워크 계층에서 내려온 데이터를 MAC 계층으로 전달하거나, MAC 계층에서 올라온 데이터를 네트워크 계층으로 전달하는 중개 역할
    - **`SAP`(Service Access Point)** 포트를 사용해 연결시켜준다(16진수)
        - SAP를 통해 데이터가 **어떤 서비스로 전달되는지 주소를 정의**
        - SAP: 특정 프로토콜 데이터가 어디로 가야 하는지 지정하는 포인트 (계층 간 데이터 전송 지점을 정의)
        - SAP를 이용한 다중화: 여러 상위 계층 서비스가 동일한 물리적 네트워크를 공유할 수 있도록 함

### ✔️ MAC (Media Access Control) 부계층

네트워크 장치가 물리적인 네트워크 매체에 접근하는 방법을 정의

- **물리 네트워크에 대한 접근 제어를 담당**
    - 네트워크 장치가 언제, 어떻게 데이터 전송을 시작할 수 있는지 제어
    - 여러 장치가 같은 네트워크를 공유하는 상황에서 충돌을 방지하기 위한 접근 제어
- **MAC 주소(Address)**
    - MAC 주소는 **네트워크 장치의 고유 식별자** → 네트워크에 연결된 모든 장치는 MAC 주소를 통해 식별됨
        - 네트워크에 연결된 각 호스트의 그룹 혹은 모든 호스트로 표현 가능
    - MAC 주소를 기반으로 데이터 패킷을 전송함
        - 프레임이 특정 장치로 전송될 때 → 수신 장치의 MAC 주소를 사용해 데이터 전달
- **매제 접근 제어: 충돌 방지**
    - 네트워크에서 **여러 장치들이 동시에 데이터를 전송하려고 할 때 발생하는 충돌을 방지**
    - CSMA/CD 충돌 방지 프로토콜 MAC 계층에서 사용

---

- **프레임 형태 인식**
    - 데이터를 전달하는 단위인 프레임의 형식을 인식 → 어떤 종류의 프레임인지 구분 가능
    - ex. 이더넷 프레임과 무선 프레임은 각각 다른 형식을 가지기 때문에 이를 구분해 처리
- **프레임 제어**
    - `프리엠블(preamble)`:
        - 프레임의 맨앞에 붙는 미리 정의된 비트 패턴
        - 이것을 통해 각각의 프레임 제어
        - 데이터 전송을 시작할 준비를 알리고, 송신자와 수신자 간 동기화를 맞추는 데 사용
    - `FCS(Frame Check Seqience; 순환 중복 검사)`
        - 데이터의 오류를 검출하기 위해 프레임에 추가되는 검증값
        - CRC(Cyclic Redundancy Check)를 통해 수신한 프레임의 에러 유무를 제어(오류가 있는지없는지 체크)
- **프레임 복사**
    - LAN 카드는 프레임을 네트워크로부터 받아 임시 메모리(버퍼 공간)에 저장한 다음, 필요한 곳으로 전송하는 과정을 거침 (Store and Forwarding; 저장하고 전송하는 방식)
    - 프레임의 모든 데이터를 확인한 후 전송 → 데이터의 정확성을 높일 수 있음

# 02 LAN의 분류

LAN의 분류 방법: 

- **토폴로지**에 의한 분류: 성형, 버스형, 트리형, 링형
- **전송매체**에 의한 분류: 광케이블, 트위스티드 페어 케이블, 동축 케이블
- **전송방식**에 의한 분류
    - **데이터 흐름**에 따른 전송방식에 의한 분류: 직렬전송방식/병렬전송방식
    - **다중화**에 따른 전송방식에 의한 분류: 주파수 분할 다중화 방식(FDM), 시분할 다중화 방식(TDM), 부호 분할 다중화 방식(CDM)
- **전송 신호**에 의한 분류: 기저대역(Baseband) 전송방식, 광대역(Broadband) 전송방식
- **매체 접근 방법**에 따른 분류: CSMA/CD, 토큰링 방식, 토큰버스 방식

## ✅ 토폴로지(Topology)에 의한 분류

네트워크에서 스테이션들을 연결하는 케이블의 구조, 방법 혹은 기하학적인 모양

다양한 방식으로 구성되는 네트워크 구성 방법에 따라 성형, 버스형, 트리형, 링형으로 구분

### ✔️ 성형 (Star Topology)

각 스테이션(station)이 허브(Hub)라고 불리는 중앙 전송 제어 장치와 점대점(Point-to-Point) 링크에 의해 접속되어 있는 형태

- 장점
    - 고장 발견이 쉽고 유지 보수가 용이함
    - 한 스테이션의 고장이 전체 네트워크에 영향을 미치지 않음
    - 한 링크가 떨어져도 다른 링크는 영향을 받지 않음
    - 확장이 용이함
- 단점
    - 중앙 전송 제어 장치가 고장나면 네트워크는 동작 불가능
    - 설치 시에 케이블링에 많은 노력/비용 소요
    - 통신량이 많은 경우 전송 지연 발생

### ✔️ 버스형 (Bus Topology)

하나의 긴 케이블이 네트워크상의 모든 장치를 연결하는 중추 네트워크의 역할을 하는 형태

- 장점
    - 설치하기가 용이함
    - 케이블에 소요되는 비용이 최소
    - 각 스테이션의 고장이 네트워크 내의 다른 부분에 아무런 영향을 주지 않음
- 단점
    - 재구성이나 결합 분리의 어려움
    - 탭에서 일어나는 신호의 반사는 신호의 질을 저하시킴
    - 기저대역(Baseband) 전송 방식을 사용할 경우 거리에 민감하여 거리가 멀어지면 중계기가 필요함
    - 송신하려는 정보 신호를 변조하여 반송파에 실어서 전송하지 않고 / 직접 전송하는 방식으로 주파수를 변경하지 않고 / 기저대역신호를 사용하여 그대로 전압이나 빛의 강도를 변환하여 전송하는 방식
    - 버스 케이블에 결함이 발생하면 전체 스테이션은 모든 전송을 할 수 없음
    - 스테이션의 수가 증가하면 처리 능력은 급격히 감소
    - 네트워크에 부하가 많으면 응답시간이 늦어짐

### ✔️ 트리형 (Tree)

- 성형의 변형
- 트리에 연결된 스테이션은 중앙 전송 제어 장치(1차 허브)에 연결되어 있지만, 
모든 장치가 중앙 전송 제어장치에 연결되어 있지 않은 형태
- 특징은 스타형과 비슷함
- 2차 허브를 위치시킴으로써 장점
    - 하나의 1차 허브에 더 많은 스테이션을 연결함
    - 각 스테이션 간의 신호의 이동 거리 증가시킴
    - Concentrator

### ✔️ 링형 (Ring)

닫힌 루프 형태로 각 스테이션이 자신의 양쪽 스테이션과 전용으로 점대점으로 연결된 형태

- 장점
    - 단순하며 설치와 재구성이 쉬움
    - 장애가 발생한 스테이션을 쉽게 찾음
    - 스테이션의 수가 늘어나도 네트워크의 성능에는 별로 영향을 미치지 않음
    - 성형보다 케이블링에 드는 비용이 적음
- 단점
    - 링을 제어하기 위한 절차가 복잡해 기본적인 지연이 존재
    - 단방향 전송이기 때문에 링에 결함이 발생하면 전체 네트워크를 사용할 수 없음 → 이를 해결하기 위해 이중링을 사용함
    - 새로 스테이션을 추가하기 위해서는 물리적으로 링을 절단하고 스테이션을 추가해야 됨

## ✅ 전송매체에 의한 분류

### ✔️ 광케이블 (Fiber Optic Cable)

- 빛에 데이터를 실어서 전송하는 전송매체
- **전자기파의 간섭에 무관**하며, 트위스티드 페어 케이블이나 동축 케이블에서 지원할 수 없는 **높은 속도**를 제공
- **철저한 보안**이 요구되는 경우에 사용
- 케이블에 스테이션을 **접속하기 어렵기** 때문에 허브, 고속의 링 또는 점대점 구성에 이용
- 공기,물과 같이 밀도가 높은 곳을 통과할 때는 속도가 감소됨
- **잡음에 대한 저항력 강함, 낮은 신호감쇠, 높은 대역폭, 고비용, 설치와 관리가 어려움**
- 전파방식: **Single mode(단파장**) vs **Multi mode(다중파장)**
- LAN에서 현재 FDDI(Fiber Distributed Data Interface)와 DQDB(Distributed-Queue, Dual-bus), 기가 비트 고속 이더넷 등에서 사용

### ✔️ 트위스티드 페어케이블 (Twisted Pair Cable)

두 줄의 도선을 쌍으로 꼬아서 만들 케이블로, **잡음에 대한 내성**을 가지고 있는 케이블 (잡음을 상쇄시키기 위해 꼬아놓음)

- 전화선, LAN선, 신용카드 결제기, 건물 내부 통신선
- **비용 낮음, 데이터율 낮음, 범위 짧음**
- **비 차폐 트위스티드 페어(Unshielded Twisted Pair; UTP)**
    - 기존의 전화 시스템에 사용되는 매체이기 때문에 별도의 **설치 비용이 들지 않음**
    - **전송 속도에 제한**이 있어 비교적 **소규모의 LAN 환경에 쓰임**
    - 총 8가닥, 4쌍
        - CATEGORY3(~16Mbps),CATEGORY4(~20Mbps),  CATEGORY5(~100Mbps),CATEGORY6(~200Mbps,~250Mbps)
- **차폐 트위스티드 페어(Shidlede Twisted Pair; STP)**
    - UTP의 **간섭과 잡음의 영향을 줄인** 것
    - **비용이 비싸고 작업하기 어려움**

### ✔️ 동축 케이블 (Coaxial Cable)

우수한 주파수 특성을 가지고 있으므로 → **높은 주파수와 빠른 데이터 전송**이 가능

- **장거리 전송용**으로 성능이 좋음
- TV선, 케이블TV선, 장거리 다회선 전화선, LAN/MAN
- **잡음에 강하며, 높은 대역폭, 높은 주파수 범위**의 반송파 지원
- 전송 신호에 따라(아래 전송 신호에 따른 분류 참고)
    - **기저대역(Baseband) 전송 방식의 동축 케이블**
        - **디지털 신호를 그대로** 전송하는 경우
        - 광대역 방식의 동축케이블에 비해 **비용이 저렴**함
        - 주파수 분할 다중화 ****방식을 이용하여 **다중 채널을 사용할 수 없음**
        - 주로 버스 토폴로지에서 사용
    - **광대역(Broadband) 전송 방식의 동축케이블**
        - **아날로그 신호로 전송**하며 해당 대역폭을 할당하여 사용
        - **주파수 분할 다중화**를 통해 **독립적인 채널**을 가짐
        - 여러 개의 빌딩 간 또는 대규모의 공장 등에서 많이 사용
        - BNC (British Naval Connector; 영국해군결합기)

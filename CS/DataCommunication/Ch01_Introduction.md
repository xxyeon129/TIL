# 01 데이터 통신

- 데이터(data)
  - 사용자 간에 합의된 형태로 표현된 정보
- 데이터 통신(data communication)
  - 전선과 같은 형태의 전송 매체를 통해 두 장치 간에 데이터를 교환하는 것
  - ‘통신을 한다’는 것은 ‘정보의 공유’를 의미
- 전기통신(telecommunication, 원거리 통신)
  - 먼 거리에서 행해지는 통신을 의미

## 데이터 통신 기본 특성 (4)

1. `전달성(delivery)`: 정확한 목적지에 반드시 원하는 장치나 사용자에게 전달되어야 한다.
2. `정확성(accuracy)`: 데이터를 정확하게 전달, 전송 도중에 **감쇠, 왜곡, 잡음** 등으로 내용이 변경된 데이터는 사용할 수 없다.
3. `적시성(timeliness)`: 데이터를 적정 시간 내에 전달(데이터가 만들어진 그대로, 만든 순서대로, 과도한 지연 없이 전송) - 실시간 전송
4. `파형 난조(jitter)`: 패킷 도착 시간이 서로 조금씩 다른 것. 음성이나 동영상 패킷 도착 시간이 서로 일정치 못함

## 데이터 통신 구성 요소 (5)

1. 메시지(Message)
   - 통신 대상인 전송되는 정보(데이터)
   - 정보는 문자, 숫자, 소리, 그림, 영상 또는 이들의 조합
2. 송신자(Sender)
   - 메시지를 보내는 장치(컴퓨터, 핸드폰 등)
3. 수신자(Receiver)
   - 메시지를 받는 장치(컴퓨터, 핸드폰, TV 등)
4. 전송매체(Medium)
   - 메시지가 송신자에서 수신자까지 메시지를 전달하는 물리적인 경로
     - 꼬임쌍선(twisted pair wire)
     - 동축케이블(coaxial cable)
     - 광케이블(fiber-optic cable)
     - 레이저 또는 무선파
5. 프로토콜(Protocol)
   - 데이터 통신을 제어하는 규칙들의 집합
   - 송/수신자는 반드시 동일한 프로토콜을 사용

## 데이터 표현 방식

- **문자(text)**: 비트 패턴 (0과 1로 된 비트들의 순차열로 표현)

  - 코드(code): 부호를 표현하기 위한 비트 패턴들의 집합
  - 코딩(coding): 부호를 표현하는 과정
  - ASCII: 7bit개의 **문자**에 해당, 기본 라틴(Basic Latin) 숫자를 나타내는데 사용되지 않는다 (숫자 → 2진수 변환)
  - 유니코드(Unicod): 32bit 사용. **전세계** **모든 문자** 표현

- **숫자(number)**: 비트 패턴을 사용해 표현

- **영상(images, 화상)**: 비트 패턴으로 표현, 가장 간단한 형태의 영상은 화소(픽셀; pixel)라는 점들의 행렬로 구성

  - 화소의 크기는 해상도에 따라 다름 (화소 수에 따라 화질과 메모리 용량 문제)
  - 흑백(1bit 또는 2bit)
  - RGB(red, green, blue) - 컬러TV 3원색
  - YCM(yellow, cyan, magenta) - cmyk(black) 출판

- **오디오(audio)**: 연속적인 신호를 소리나 음악을 기록하는 것

  - 소리나 음악을 마이크로폰을 사용하여 전기신호로 변환할 때 연속신호를 사용

- **비디오(video)**: 연속적인 개체 또는 여러 화상의 조합

## 데이터 전송 방식

연결된 두 장치 간에 데이터를 전송할 때 신호가 전달되는 방향을 기준으로 구분

### 단방향 방식 (simplex mode)

- 한 방향으로만 통신이 가능
- 두 장치 간 한쪽은 전송만 가능하고, 다른 쪽은 수신만 가능
- ex. 자판은 입력만, 모니터는 출력만 가능

### 반이중 방식 (half-duplex mode)
양방향으로 데이터가 전송될 수 있지만, 동시에 전송하는 것은 불가능한 방식

- 각 지국은 송신과 수신이 가능하지만, 동시에는 불가능
- 장치가 송신하면 다른 장치는 수신만 가능
- ex. 워키토키, 생활 무전기

### 전이중 방식 (full-duplex mode)
하나의 전송 선로에서 데이터가 동시에 양방향으로 전송될 수 있는 방식

- 양쪽 지국이 동시에 송신과 수신이 가능
- 신호는 링크의 용량을 공유해서 양방향으로 전달
- ex. 전화

# 02 네트워크

## **네트워크 (network; 망)**

서로 연결된 장치들이 데이터를 주고받을 수 있는 통신 시스템

여러 호스트들이 연결장치(라우터, 스위치 등)와 채널을 통해 데이터를 주고받는 시스템

### 네트워크의 주요 구성 요소

**`호스트`**(host, 종단 시스템(end system))”

- 네트워크에 연결된 실제 장치들 (컴퓨터, 스마트폰, 서버 등 데이터를 주고받는 모든 기기가 호스트)
- 대형 컴퓨터, 데스크톱, 랩톱, 휴대폰, 보안 시스템

### 연결장치 (3)

호스트들이 서로 데이터를 주고받을 수 있도록 연결하는 장치

- **라우터(Router):** 데이터가 이동할 경로를 설정해주는 장치. 서로 다른 네트워크를 연결하는 장치(ex. 집의 네트워크를 인터넷에 연결)
- **교환기(Switch):** 네트워크 내의 호스트들끼리 데이터를 주고받을 수 있도록 연결해주는 장치
- **모뎀(변/복조기; Modem):** 데이터의 형태를 변환하는 장치
    - 인터넷 신호와 컴퓨터의 신호를 변환해주는 장치(ex. 인터넷 신호 아날로그 신호 → 디지털 신호로 바꿔 사용할 수 있도록 함)

### **채널 (channel)**

데이터를 주고받을 때 사용하는 경로. 

- 네트워크에서 **장치들이 연결되는 링크**를 의미
    - 링크: 네트워크의 물리적/논리적 연결. 데이터를 주고받기 위해 직접 연결된 경로
- 장치들은 **유선 또는 무선 통신** 매체를 통하여 연결
- 채널의 유형
    - 유선(Wired): 케이블, 광섬유 같은 물리적인 선을 이용해 데이터를 주고받는 방식 (ex. 이더넷 케이블을 꽂아 유선 채널을 통해 인터넷 연결)
    - 무선(Wireless): 전파, 빛 같은 비물리적인 매체를 이용해 데이터를 주고받는 방식 (ex. WiFi, 블루투스)

## 네트워크 평가 기준 (Network Criteria)

### 1️⃣ 성능(Performance)

전달시간이나 응답시간 등 여러 방법으로 측정

- 전달 시간(transit time): 메시지가 한 장치에서 → 다른 장치로 이동하는 데 걸리는 시간
- 응답 시간 (response time): 요청과 응답에 걸리는 시간

네트워크 성능은 여러 요인에 따라 달라짐

- 사용자 수
- 전송 매체 유형 (데이터 전송률)
- 연결된 하드웨어
- 소프트웨어의 효율성

처리율(throughput, 또는 처리량)과 지연(delay) 두 가지 척도로 평가

### 2️⃣ 신뢰성 (Reliability)

- 고장 빈도 수
- 고장이 난 후 링크를 복구하는데 소요되는 시간
- 재난 발생 시 네트워크의 안전성 등에 의해 측정

### 3️⃣ 보안 (Security)

- 불법적인 접근으로부터 데이터 보호
- 손상으로부터 데이터 보호
- 개발, 정책 구현
- 침해나 데이터 손실로부터 복구 절차 등을 포함

## 물리적인 구조: 연결 유형

### 점-대-점 (point-to-point) 연결

- **두 장치가 하나의 전용 링크로 직접 연결**되는 방식
    - 두 장치만 링크를 사용하므로, 링크를 통해 오가는 데이터는 오직 이 두 장치만 사용할 수 있음
- **채널의 전체 용량**은 **두 장치 간 데이터 전송을 위해서만** 사용
    - ✨ 데이터 **충돌이나 간섭이 발생할 가능성 거의 없음**
    - 💩 연결된 장치가 많아질수록 장치 간에 필요한 **케이블 수가 늘어남.**
    - 💩 네트워크가 복잡해지고 **설치 비용 증가**
- 유선 연결이 주로 사용됨 → 양쪽 끝에 연결된 케이블이나 전선 사용
- ex. 텔레비전과 리모컨 사이의 연결, 컴퓨터와 프린터를 테이블로 직접 연결하는 경우 (다른 기기들과 신호를 공유하지 않음)

### 다중점(multipoint, 멀티드롭;multidrop) 연결

- 하나의 링크를 3개 이상의 장치들이 공유하는 방식
    - 여러 장치가 같은 링크에 연결되어 있음 → 각 장치는 링크의 용량을 나눠서 사용
    - 데이터가 한 장치에서 링크를 통해 여러 장치로 전송될 수 있음
- **채널의 용량을 공간적으로 또는 시간적으로 공유**
    - 공간적으로 공유: 여러 장치가 동시에 링크를 사용하지만, 링크의 전체 용량을 나눠서 사용 → 동시에 여러 장치가 데이터를 전송할 경우 데이터 충돌 위험
    - 시간적으로 공유: 여러 장치가 시간을 나눠서 순차적으로 링크를 사용하는 방식 → 충돌 방지
- ✨ 케이블 **비용을 줄일 수 있음**
- 💩 데이터 **충돌 위험 → 충돌 방지 기법 필요**
- 💩 링크의 용량이 분산되기 때문에, 점대점 방식에 비해 **전송 속도가 느림**

## 물리적인 구조: 물리적인 접속 형태

- 토폴로지 (physical topology)
- 네트워크에서 (컴퓨터의 위치나 컴퓨터의 케이블 연결 등) 물리적인 / 논리적인 배치 방식

### 1️⃣ 그물형 접속형태 (mesh topology)

- 모든 장치는 다른 장치에 대해 전용의 점-대-점 링크를 가짐
- 전용 의미는 연결되어 있는 두 장치 간 통신만 담당하는 링크가 있음
- 링크 수는 각 노드 (n) 당 (n-1)개로 n(n-1) 물리 링크가 필요
- 각 링크가 전이 중 모드(양방향 통신)를 허용하면 링크 수는 n(n-1)/2개 필요
- ✨ 장점
  - 전용 링크의 사용으로 원하는 자료의 전송을 보장
  - 안정성이 높음
  - 프라이버시와 보안
- 💩 단점
  - 설치와 재구성이 어려움
  - 큰 설치 공간과 높은 비용

### 2️⃣ 성형 접속형태 (star topology)

각 스테이션이 허브(Hub)라고 불리는 중앙제어장치와 점대점 링크에 의해 접속되어 있는 형태

- 각 장치들은 서로 직접 연결되어 있지 않다.
- 허브가 각 장치 간의 교환 역할 (스위치 역할)
- 각 장치 간 직접적인 통신 불가, 모든 전송은 제어 장치를 통해 전송
- 각 장치와 연결을 위해 1개의 링크와 1개의 I/O 포트만 필요
- ✨ 장점
  - 한 링크의 문제가 다른 링크에 영향을 주지 않음
  - 설치, 재구성, 확장이 용이
  - 고장 발견이 쉽고 유지보수가 용이함
- 💩 단점
  - 허브가 고장 나면 전체 시스템 고장
  - 설치 시 (링형, 버스형보다) 많은 케이블 연결, 비용이 필요
  - 통신량이 많은 경우 전송지연 발생

### 3️⃣ 버스형 접속형태 (bus topology)

하나의 긴 케이블이 네트워크상의 모든 장치를 연결하는 중추 네트워크 역할을 하는 형태

- 다중점 형태
- 하나의 긴 케이블이 모든 장치를 연결하는 백본 또는 중추 네트워크 역할
- 탭(tap)과 유도선(drop line)에 의해 버스에 연결

(브로드캐스트 방식으로 전달)

- ✨장점
  - 설치가 쉽다
  - 가장 적은 양의 케이블 사용  → 케이블에 소요되는 비용이 최소
  - 각 스테이션(장치)의 고장이 네트워크 내의 다른 부분에 영향을 주지 않음
- 💩 단점
  - 재구성이나 결함 분리의 어려움
  - 중추 케이블의 결함 시 전체 장치에 영향을 줌
  - 기저대역(Baseband) 전송 방식을 사용할 경우 → 거리에 민감하여 거리가 멀어지면 `중계기`가 필요
    - 기저대역 전송방식: 송신하려는 신호를 변조해 반송파에 실어서 전송하지 않고, 직접 전송하는 방식 → 주파수를 변경하지 않고 기저대역신호를 사용해 → 그대로 전압이나 빛의 강도를 변환하여 전송하는 방식
  - 장치의 수가 증가하면 → 처리 능력 급격히 감소
  - 네트워크에 부하가 많으면 → 응답 시간 늦어짐

### 4️⃣ 링형 접속형태 (ring topoloty)

닫힌 루프 형태로, 각 장치가 자신의 양쪽 장치와 전용으로 점-대-점 연결으로 연결된 형태

- 각 장치는 중계기(repeater; 신호재생기) 포함
- 일정한 시간을 보장해줌
- ✨ 장점
  - 설치와 재구성이 쉽다
  - 장애가 발생한 장치를 쉽게 찾음
  - 장치의 수가 늘어나도 네트워크의 성능에는 별로 영향을 미치지 않음
  - 성형보다 케이블링에 드는 비용이 적음
  - 신호는 항상 순환되므로, 만약 한 장치가 특정한 시간 내에 신호를 받지 못하면 경보를 발생함
- 💩 단점
  - 단방향 전송 → 링에 결함이 발생하면 전체 네트워크 마비
    - `이중 링` 또는 결함이 있는 지점을 단절시킬 수 있는 교환기를 사용해 해결
- 링을 제어하기 위한 절차가 복잡하여 기본적인 지연이 존재
- 새로 장치를 추가하기 위해서는 물리적으로 링을 절단하고 장치를 추가해야 함

### 5️⃣ 혼합형 접속형태 (hybrid topology)

broad casting(방송) 방식으로 패킷 전송

### 6️⃣ 트리형 접속형태 (Tree Topology)

성형의 변형 → 트리에 연결된 스테이션은 중앙 전송 제어 장치(1차 허브)에 연결되어 있지만, 모든 장치가 중앙 전송 제어장치에 연결되어 있지 않은 형태

- 특징은 성형과 비슷함
- 2차 허브를 위치시킴으로써 장점
    - 하나의 1차 허브에 더 많은 스테이션을 연결함
    - 각 스테이션간의 신호와 이동 거리를 증가시킴
    - Concentrator

# 03 네트워크 유형

땅에 묻혀 있기 때문에 네트워크 유형 구분이 어렵다. → 따라서 크기, 지리적인 범위, 소유권으로 구분한다.

## 근거리 통신망(LAN)

- Local Area Network
- 개인 소유, 단일 사무실, 건물, 학교 등에 있는 호스트들을 연결
- 각 호스트는 LAN에서 호스트를 유일하게 구분하는 식별자 주소를 가짐

- 과거 패킷 전송 방식: 모든 호스트는 한 호스트에서 보낸 패킷을 → 모든 호스트가 받을 수 있는 공통 케이블로 연결되어, 해당 수신자는 패킷을 받고 다른 호스트는 패킷을 폐기
- 오늘날 패킷 전송 방식: LAN은 패킷의 목적지 주소를 인식할 수 있어 해당 목적지로만 패킷을 전달하는 성능이 좋은 교환기 사용
  - 교환기는 LAN에서 트래픽을 조절하고, 서로 같은 발신지와 목적지가 아니면 같은 시간에 한 쌍 이상이 통신하도록 해준다.

> [!NOTE] > **IP**
>
> - 공인(서버): 고정 IP(Static IP)
> - 비공인(X): 내부 IP 주소. 유동 IP(Dynamic IP)

## 광역 통신망 (WAN)

- Wide Area Network
- 넓은 지리적인 영역을 갖는 도시나 주, 국가, 또는 전세계에서 사용
- 교환기, 라우터(패킷의 전송 최적 경로를 지정해줌), 모뎀(MODEM; 변조;전송하기 적합한 신호로 바꿔줌)과 같은 장치를 사용하여 서로 연결
- 통신 회사가 만들고 이를 임대하기 위한 목적으로 사용
- 사설망(private network): 한 기관이 소유하고 관리되면서 그 기관에 의해서만 사용하는 WAN
- 공중망(public network): 한 기업이 소유하지만 일반인들에게 ‘빌려주는’ WAN
  - 공중전화망

### 점-대-점(point-to-point) WAN

- 전송매체를 통해 두 통신 장치를 연결하는 네트워크

### 교환형 WAN

- 2개보다 더 많은 종단점을 갖는 네트워크
- 오늘날 전세계적인 통신 Back-bone망에 사용
- 교환기에 의해 연결된 여러 개의 점-대-점 WAN

## 혼합형 네트워크

4개의 WAN과 3개의 LAN으로 이루어진 혼합형 네트워크

## 교환 (switch)

- 인터넷
  - 적어도 2개의 링크를 연결하는 교환형 네트워크(switched network, 또는 교환망)
  - 교환기는 한쪽 링크에서 다른 링크로 데이터를 포워딩(forwarding;발송)해야 한다.
- 회선 교환망(circuit-switchde network)
  - 두 종단 시스템 사이에 회선(circuit)이라는 전용선을 이용
  - 교환기: 회선의 활성화와 비활성화시키는 기능
  - 과거 전화: 회선교환망, 현재는 패킷 교환망
  - 대용량 회신은 동시에 4개 음성통신 가능 ( 동시에 4:4 음성통신 가능)
  - 모든 회선이 다 사용하지 않은 때는 비효율적

## 패킷 교환망

요즘은 회선교환 잘 안쓰고 패킷을 주고받는다.

- 컴퓨터 네트워크에서 두 종단 시스템 사이에 **패킷(packet)이라는 블록**을 이용
- 교환기가 포워딩과 저장 기능으로 패킷을 저장 후 나중에 포워딩
- 라우터는 패킷을 저장하고 포워딩하기 위해 큐(queue)를 소유
- ex) 고용량 회선이 저용량 회선의 두 배일 때, 두 컴퓨터 간 통신 시 패킷을 기다릴 필요가 없다. 그러나 고용량 회선이 모두 사용되고 있는 상태에서 패킷들이 라우터에 도착하면 라우터는 이 패킷들을 저장하고 도착한 순서대로 포워딩해야 한다.
- 회선 교환망보다 효율적이지만 **패킷 지연 문제**가 있다.

## 인터넷

internet (소문자): 서로 통신할 수 있는 둘 또는 그 이상의 네트워크 집합

Internet (대문자): 전세계적으로 널리 사용되고 있는 TCP/IP를 사용하고 있는 특정 네트워크

- 인터넷 서비스 제공자(ISP; Internet Service Provider)
  - 백본망: Sprint, Verizon(MCI), AT&T, 그리고 NTT와 같은 통신 회사들이 소유하고 있는 거대한 네트워크
- 제공자 네트워크: 요금을 지불하고 백본의 서비스를 이용, 백본에 연결되며, 다른 제공자 네트워크와도 연결
- 소비자 네트워크: 인터넷의 종단에 위치, 실질적으로 인터넷에서 제공되는 서비스를 위해서 제공자 네트워크에게 요금을 지불

### 오늘날의 인터넷

- 대등점(peering point): 복잡한 교환 시스템에 의해 연결
- 백본망: 주로 국제 인터넷 서비스 제공자
- 제공자 네트워크: 주로 국가 또는 지구 인터넷 서비스 제공자

### 인터넷 접속

오늘날 인터넷은 네트워크 간 상호 연결이며, 사용자들이 그 일부가 되게 한다.

물리적으로 ISP에 연결(점-대-점 WAN)되어야 한다.

- **전화망(telephone network) 이용**
  - 전화정보서비스(dial-up service): 음성 전화선에 모뎀을 설치하여 데이터 통신이 가능하게 함
  - 인터넷 속도가 매우 느림
  - 디지털 가입자 회선(Digital Subscriber Line): 전화선을 이용한 음성과 데이터 통신이 동시에 가능
  - ADSL(Asymmetric Digital Subscriber Line: 비대칭 디지털가입자회선) 8M
  - VDSL(Very High Rate Digital Subscriber Line: 초고속 디지털 가입자 회선) 12~50M
- **케이블 TV망 이용**
- **무선망 이용:** 무선 LAN + 유선 LAN
- **인터넷에 직접 연결:** 큰 기관이나 회사가 하나의 지역 ISP가 되어 인터넷에 직접 연결
  - 기관이 ISP로부터 고속 WAN을 임대하여 지역 ISP에 연결

# 05 프로토콜과 표준

## 프로토콜 (Protocol)

- 개체 간에 이루어지는 데이터 통신을 제어하는 규칙들의 집합

**주요 요소 (3)**

- **구문(Syntax)**
  - “무엇을?”에 해당
  - 데이터 구조와 형식, 부호화, 크기
  - 데이터가 어떤 순서로 표현되는가에 관한 내용
- **의미(Semantics)**
  - “어떻게?”에 해당
  - 데이터 형태에 대해 어떻게 해석하고, 어떤 동작을 취할 것인가?
  - 전송 제어 및 오류 처리를 위한 제어 정보 등을 포함
- **타이밍(Timing)**
  - “언제?”에 해당
  - 언제 데이터를 전송해야 할 것인가와 얼마나 빠른 속도로 전송할 것인가?
  - 수신자와 송신자 간의 데이터 전송 시간과 속도 조절

## 표준(Standard)

- 사회 이익 증진을 목적으로 과학기술 및 경험의 종합적인 결론이나 이해 관계자의 협력과 모든 의견, 대다수의 승인에 의해서 작성된 기술 규격서 또는 그 외의 문서
- 통신기기 시장을 개방적이고 경쟁적으로 만들기 위한 관리 문서
- 국제 또는 국내의 데이터 통신 기술 및 처리에 대한 호환성 보장

- **표준의 분류**
  - `법률 표준`(by law): 공식적으로 공인된 기관에 의해 입법화된 표준
  - `사실 표준`(by fact): 신제품이나 신기술의 기능을 규정한 제조업체에 의해 제정된 표준
    - `특허(proprietary)/폐쇄(closed) 표준`: 영리기관이 자사 제품의 기본 원리를 창안한 것으로 독점적 권리 소유
    - `비특허(non-proprietary)/개방(open) 표준`: 집단이나 위원회에서 개발하여 공개한 표준

### 국제 표준

- 1850년대 이후 필요성 대두
- 지역적 한계를 극복하고 국제 교류를 활성화하기 위함

- **국제 표준화 기구(ISO)**

  - International Standards Organization
  - 각국의 표준제정위원회에서 선정된 위원들로 구성된 다국적 기구
  - 수행 업무
    - 표준 및 관련 활동의 세계적인 조화를 촉진
    - 국제표준의 개발, 발간, 세계적인 보급
    - 회원기관 및 기술위원회의 작업에 관한 정보 교환의 주선
    - 표준 관련 사안에 관한 다른 국제기구와의 협력

- **국제전기통신연합(ITU-T)**

  - International Telecommunications Union
  - CCITT(Consultative Committee for International Telegraphy and Telephony) 후신
  - 일반 전기통신, 전화, 데이터 통신 시스템 표준
  - 1947년 UN 산하기관으로 창설

- **미국 국립 표준원(ANSI)**

  - American National Standards Institute
  - 미국의 표준안을 제정하는 사단법인, ISO의 미국 대표단체
  - 미국 및 미국 시민의 복리 증진을 위해서 수행
  - 미국 내 임의 표준의 국가 조정 기구로서 역할
  - www.ansi.org

- **전기전자공학회(IEEE)**

  - Institute of Electrical and Electronics Engineers
  - 세계에서 가장 규모가 큰 전문공학 학회
  - 전기, 전자, 전기통신, 컴퓨터 분야의 전문 공학 학회
  - 전기 공학, 전자 공학, 무선 공학, 컴퓨터 공학 및 공학의 제반 관련 분야의 학술 증진, 창안, 제품 품질 제고
  - www.ieee.org

- EIA; Electronic Industries Association (미국 전자 산업 협회)

  - 미국의 전자기기 제조업체를 대표하는 단체
  - 전자기기 관련 산업을 촉진하기 위해 설립된 비영리 기관

- NIST; National Institute Standards and Technology(미국 국립 표준 기술원)
  - 1901년에 미국 상무부에서 창설된 미국 표준 기관
  - 경제 분야의 보안을 강화하고 삶의 질을 개선하는 방법으로 측정 과학, 표준 및 기술을 발전하여 미국의 혁신과 산업경쟁력을 촉진

### 포럼(Forum)

사회자 지도 하에 한 사람/여러 사람이 연설하고, 그에 대해 청중이 질문하면서 토론을 진행하는 방식

- 기업과 대학, 사용자와 공동 작업을 하여 신기술을 시험하고, 평가하며, 표준을 제정
- 광대역(Broadband) 포럼
- 인터넷(Internet) 포럼

### ✔️ 인터넷 표준(internet standard)

**인터넷에서 통신할 때 반드시 지켜야 하는 규칙이나 규격**

인터넷 표준이 만들어지기 위해서는 여러 단계의 절차를 거친다. 처음부터 바로 표준으로 인정받는 게 아니라, 비평과 논의를 통해 점차 개선되면서 표준이 완성됨 → 이 과정은 RFC라는 문서를 통해 이루어짐

- RFC(Request for Comment)
  - 비평을 기다리는 문서
  - 공학 분야에서 새로운 연구, 혁신, 기법 등을 포함하는 문서
- 완성 단계를 거친 후 → 요구수준에 따라 분류

**[인터넷 표준의 형성과정]**

- `Internet draft`로 시작하여 → 상태에 따라 완성된 처리 절차를 가짐
  - Internet draft: 6개월의 유효기간을 갖는 작업문서
- **`제안 표준`**(proposed standard):
  - **안정성이 있는 규격**
  - 인터넷 공동체의 검토, 논의를 거쳐 표준이 될 가능성이 높은 상태
- **`드레프트 표준`**(draft standard):
  - 적어도 2번의 독자적인 성공 + 상호 운용성이 이루어진 규격
  - 문제점이 발생하면 수정하고, 문제점이 없다면 통상적으로 인터넷 표준으로 발전
- **`인터넷 표준`**(internet standard): **모든 구현이 완전히 이루어진 규격**
  - 모든 인터넷 시스템에서 준수해야 할 규칙이 됨

---

**[기타 표준의 상태]**

모든 표준이 최종적으로 인터넷 표준이 되는 것은 아니다. 어떤 것들은 다른 단계에서 멈추거나 특별한 상태로 분류될 수 있다.

- 기록 단계(historic):
  - 인터넷 표준이 되기 위한 단계를 통과하지 못한 규격
  - 역사적인 면에서 중요한 의미를 가짐
- 실험 단계(experimental):
  - 인터넷 운영에 영향을 주지 않는, 실험적인 작업 규격
  - 인터넷 서비스 기능으로 구현되지 않을 수도 있음
- 정보 제공(informational): 인터넷과 관련된 일반적이고 역사적인 튜토리얼 정보를 제공하는 문서

---

**[요구 수준]**

인터넷 표준은 중요도에 따라 요구 수준이 분류됨

1️⃣ 요구(required)

- 모든 인터넷 시스템에서 반드시 구현해야 하는 규격
- ex. IP(인터넷 프로토콜), ICMP(인터넷 제어 메시지 프로토콜)

2️⃣ 권고(recommended)

- 최소한의 적합성이 요구되지 않지만, 유용성이 있기 때문에 권고
- ex. FTP(파일 전송 프로토콜), TELNET
  3️⃣ 선택(selective)

- 요구되지도, 권고되지도 않는 등급
- 시스템에 유익할 경우 사용

4️⃣ 사용 제한(limited use)

제한된 상황에서만 사용(실험적인 RFC)

5️⃣ 미권고(not recommended)

일반적인 용도에 적합하지 않은 것(더이상 사용되지 않는 기록 RFC)

### ✔️ 인터넷 관리 조직

1️⃣ **ISOC** (Internet Society)

- 1992년 국제적인 비영리단체로 **인터넷 표준 제정**을 지원하기 위해 결성
- **인터넷과 관련된 학술적인 활동, 연구** 담당
- IAB, IETF, IRTF, IANA와 같은 **행정상의** 다른 인터넷 단체에 대한 **관리, 지원**

2️⃣ **IAB** (Internet Architecture Board; 인터넷 구조 위원회)

- ISOC를 위한 기술적인 자문위원회
- **인터넷 프로토콜(TCP/IP)과 관련된 기술 개발 감독**
- 인터넷 공동체의 연구원에게 **기술적인 조언** 제공

3️⃣ IETF (Internet Engineering Task Force; 국제 인터넷 표준화 기구)

- **인터넷 구조 개선**, **원활한 운영**을 위해 조직
- 인터넷 운영상의 **문제점을 파악, 해결책 제공**
- **인터넷 표준 규격 개발, 검토**
- 인터넷 운영상 문제점 파악, 해결책 제공
- IESG(Internet Engineering Steering Group)에 관리되는 작업그룹
- 현재 7개의 영역
  - Applications and Real-Time Area(art)
  - General Area(gen)
  - Internet Area(int)
  - Operations and Management Area(ops)
  - Routing Area(rtg)
  - Security Area(sec)
  - Transport Area(tsv)
  - IESG(Internet Engineering Steering Grop)에 의해 관리되는 작업 그룹의 포럼

4️⃣ IRTF (Internet Research Task Force)

- 인터넷 프로토콜의 응용, 구조, 기술과 관련된 **장기 연구주제**를 다룸
- IRSG(Internet Research Steering Group)에 의해 관리되는 작업 그룹의 포럼

> **요약**
>
> - **ISOC**: 인터넷 표준 제정을 위한 **비영리 조직**, 여러 인터넷 관련 단체를 **지원 및 관리**.
> - **IAB**: **인터넷의 기술 구조를 감독**하고, 표준 개발에 **기술적 자문** 제공.
> - **IETF**: **인터넷 표준을 개발**하고, 운영 문제를 해결하기 위한 **작업 그룹 포럼**.
> - **IRTF**: **인터넷 기술의 장기 연구**를 담당하는 조직, **미래 지향적 연구**를 진행
